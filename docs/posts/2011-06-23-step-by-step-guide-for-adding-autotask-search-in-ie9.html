Step-By-Step guide for Adding AutoTask search in IE9

2011-06-23
autotasksearch

Some of you may already have found the rather useful search page here: https://ww4.autotask.net/Home/Global_Notes_Search/search.asp which allows you to do full body searches inside of ticket time entries and the like (very useful for finding serial numbers and third party ticket references where it the information isn't necessarily going to be in the main ticket summary, etc)

If you want to take it to the next level you can add it as a custom search engine. This feature is quite obvious in IE8 (http://www.microsoft.com/windows/ie/searchguide/en-en/default.mspx) but in IE9 it’s not quite so obvious how to do it- so here’s the Step-By-Step guide for Adding AutoTask search in IE9

  1. Go to: http://ieaddons.com/en/createsearch.aspx
  2. Paste in the text below into the URL box and give your search a name (e.g. autotask):
  3. https://ww4.autotask.net/Home/Global_Notes_Search/searchBody.asp?txtSearchFor=TEST&forClient=&selNoteType=-1&selEntity=2&selEntity=4&selEntity=1&selEntity=5&selEntity=6&selEntity=7&selEntity=8&selEntity=9&selPostedBy=-1&fromDatePosted=&toDatePosted=&showCriteria=1&forClientID=&projectList=&isCRM=no
  4. Click install search provider - if you then type text into the search/address bar you can click on the AutoTask button to perform a search within AutoTask.

The only downside is you have to be logged in to search (not the end of the world mind)! You may also need to amend the URL to ww1.autotask.net for America, etc. depending on your Autotask login region- you can work that out by checking the URL you get when you're logged into Autotask!
