Autotask Chrome Custom Searches

2012-04-30
aautotaskchromesearchtips

I'm not sure if this is of any use to anyone out there who uses Autotask, but I put a quick video tutorial on how to add a custom search engine to Google Chrome to search for ticket text straight from the address bar here: http://www.youtube.com/watch?v=qOfsfjZ8Z4M&feature=youtube_gdata

The only gotcha's we've seen are:

  • You need to be logged into Autotask in Chrome!
  • No menu pane (sorry!)
  • You need to change the URL used depending on your region- so http://ww4.Autotask.net/serviceDesk/TicketsList/search_requests.asp?TopNavSearch=1&Action=Generate&nc=1&searchtext=%s works for the UK (ww4), but you may need to change in other areas, e.g.
    http://ww1.Autotask.net/serviceDesk/TicketsList/search_requests.asp?TopNavSearch=1&Action=Generate&nc=1&searchtext=%s

...But for thirty seconds worth of time to set up it can save a lot of time when searching for something! 

Equally (and especially useful for us, since it searches both active and archived projects) is the same search but with https://ww4.Autotask.net/projects/views/listview/projWinBot.asp?projectnumber=%S as the search - to quickly search for a PROJECT NUMBER (same caveats as above apply)!
