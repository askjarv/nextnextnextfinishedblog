Some useful data/options exist in Tripwire IP360 that aren't easy to report on, so I've recently put together the (PowerShell) script below that leverages IP360 Commander and some direct API calls via PowerShell to give detailed CSV vulnerability reports

```
function Disable-SSLValidation {add-type @"    using System.Net;    using System.Security.Cryptography.X509Certificates;    public class TrustAllCertsPolicy : ICertificatePolicy {        public bool CheckValidationResult(            ServicePoint srvPoint, X509Certificate certificate,            WebRequest request, int certificateProblem) {            return true;        }    }"@[System.Net.ServicePointManager]::CertificatePolicy = New-Object TrustAllCertsPolicy}function Get-IP360VulnerabilityReport{    <#    .SYNOPSIS    Runs an IP360 Commander report to show vulnerability results    .DESCRIPTION    Runs an IP360 Commander report to show vulnerability results.      This script is designed to capture status information of IP360 DP's       THIS SCRIPT IS PROVIDED "AS IS" WITHOUT WARRANTY OF ANY KIND, EITHER EXPRESSED     OR IMPLIED, INCLUDING BUT NOT LIMITED TO THE IMPLIED WARRANTIES OF MERCHANTABILITY AND/OR     FITNESS FOR A PARTICULAR PURPOSE, AND/OR NONINFRINGEMENT.    This script is not supported under any Tripwire standard support program or service.     The script is provided AS IS without warranty of any kind. Tripwire further disclaims all    implied warranties including, without limitation, any implied warranties of merchantability    or of fitness for a particular purpose. The entire risk arising out of the use or performance    of the sample and documentation remains with you. In no event shall Tripwire, its authors,    or anyone else involved in the creation, production, or delivery of the script be liable for     any damages whatsoever (including, without limitation, damages for loss of business profits,     business interruption, loss of business information, or other pecuniary loss) arising out of     the use of or inability to use the sample or documentation, even if Tripwire has been advised     of the possibility of such damages. This script should be ran in the same path as the IP360Cmd.exe binary.    Example output:    "scan_time","hostname","cve","ip","vulnerability_name","cvssv3_score","vulnid","description"    "2023-08-06 14:03:35.487913+00:00","MyHostName","[CVE.69643]","192.168.31.100","MS-2023-Mar: Remote Procedure Call Runtime Remote Code Execution Vulnerability II","9.8","556732","DESCRIPTION: Windows Remote Procedure Call is subject to a remote code execution vulnerability"    Note that description is a multi-line field, contained with a quote which most CSV file handlers (such as Microsoft Excel) should work well with    #>    param([string]$OutputFile,[int]$CVSSSeverityThreshold,[int]$PreviousMonths,[parameter(mandatory)]$VNE,[parameter(mandatory)]$VnEUser,[parameter(mandatory)]$VnePassword)    # Check user input    if(!$OutputFile){$OutputFile = ".\Report.csv"; Write-Warning "No output file specified, output will be to local path report.csv"}    if(!$CVSSSeverityThreshold){$CVSSSeverityThreshold = "9"; Write-Warning "No CVSS minimum severity specified, defaulting to 9.0"} # Indicates the minimum threshold     if(!$PreviousMonths){$PreviousMonths = 1; Write-Warning "No time period specified as previous months, defaulting to 1"}    # Generate a vulnerability report    Write-Host "Gathering IP360 vulnerability info" -ForegroundColor Green    if(Test-Path .\ip360cmd.exe){        .\ip360cmd db query "SELECT scan_time As Scan_Time, nc_host.name as Hostname, nc_vuln_cve as CVE, ip_address as IP, nc_vuln.name as Vulnerability_Name, cvssv3 as CVSSv3_Score, nc_vuln.vuln_id as VulnID,nc_vuln.description As Description FROM public.nc_vuln_result INNER JOIN public.nc_vuln ON public.nc_vuln_result.vuln_id = public.nc_vuln.vuln_id INNER JOIN public.nc_host ON public.nc_vuln_result.host_id = public.nc_host.host_id LEFT JOIN public.nc_vuln_cve ON public.nc_vuln.vuln_id = public.nc_vuln_cve.vuln_id where cvssv3 > $CVSSSeverityThreshold  and scan_time >= date_trunc('month', current_date - interval '$PreviousMonths' month)" --format csv -o tempreport.csv        $Results = Import-Csv tempreport.csv -Delimiter "|"        $Results | ForEach-Object {            # Get additional vuln details to further enrich the report            $vuln = $_.vulnid            $VulnExtraDetails = Get-IP360VulnerabilityInformation -VulnID $vuln -VnE $VNE -VneUser $VnEUser -VnEPassword $VnePassword            $CVEs = $NULL            $CVES = [system.String]::Join(" ", $VulnExtraDetails.cves )            if($VulnExtraDetails.cves){$_.CVE = $CVES}            Remove-Item tmp.csv -ErrorAction SilentlyContinue            # Write-Debug "Got extra vuln data"        }        Remove-Item .\tempreport.csv -ErrorAction SilentlyContinue        $Results | export-csv $OutputFile -NoTypeInformation        Write-Host "IP360 vulnerability info report is here: $OutputFile" -ForegroundColor Green    }else{"IP360cmd.exe not found - please ensure this script is ran with IP360Cmd present"}}function Get-IP360VulnerabilityInformation{    param([parameter(mandatory)]$VulnID,[parameter(mandatory)]$VnE,[parameter(mandatory)]$VneUser,[parameter(mandatory)]$VnEPassword)    $uri = "https://$Vne/rest/v1"    $credPair = "$($VneUser):$($VnEPassword)"    $encodedCredentials = [System.Convert]::ToBase64String([System.Text.Encoding]::ASCII.GetBytes($credPair))    $headers = @{ Authorization = "Basic $encodedCredentials" }    Write-Debug "Connecting to get further vulnerability info from $uri/aspl/$VulnID"    try{Invoke-RestMethod -Uri "$uri/aspl/vulns/$VulnID" -Method Get -Headers $headers}catch{Write-Debug "Failed to get vuln data from API"}}
```

## Example Usage  

Disable-SSLValidation  
_Get-IP360VulnerabilityReport -CVSSSeverityThreshold 7 -PreviousMonths 1 -VNE "YourVnE" -VnEUser "youruser@tripwire.com" -VnePassword "YourVNEPassword"_  
or, if you've got a valid SSL certificate, you can skip the disable-ssl validation.  
_Get-IP360VulnerabilityReport -CVSSSeverityThreshold 9 -PreviousMonths 1 -outputfile "C:\\report.csv" -vne "yourvnehostname" -vneuser "youseruser@ip360.com" -vnepassword "yourpasswordhere"_  
will provides a report on discovered vulnerabilities with a CVSSv3 score of 9 or greater, detected during a scan within the last month, outputted to a file at C:\\report.csv  

Note that IP360 Commander should be setup and running successfully prior to using this - please refer to the Tripwire IP360 Commander Implementation Guide to ensure authentication and access is configured appropriately. This script should be ran in the same path as the IP360Cmd.exe binary.  
**Example output**

"scan\_time","hostname","cve","ip","vulnerability\_name","cvssv3\_score","vulnid","description"
